MU Signature Mixin

The .mu-sig() mixin has a set of variables that can be passed to it to define size, style and printing options. This is also useful in resetting options for media queries. Below are the mixin options.

@logo-size:         "64"  
@logo-size:         "48"
@logo-size:         "32" // Default
@logo-size:         "24"

@logo-style:        "stroked" // Default 
@logo-style:        "none"

@wordmark-color:    "black" // Default 
@wordmark-color:    "white"

@print:             "no" // Default
@print:             "yes"

@center:            "no" // Default
@center:            "yes"

Dark background example with options

The MU logo with a stroke around it should be used on signatures with a colored background. See section 1.2 of the Graphic Identity Standards at identity.missouri.edu.

Mizzou Logo

University of Missouri

<div class="mu-sig-dark-example">
    <p class="logo"><a href="http://missouri.edu">Mizzou Logo</a></p>
    <h1 class="wordmark"><a href="http://missouri.edu">University of Missouri</a></h1>
</div>
.mu-sig-dark-example {
    padding: 1.5em;
    background: #333;
    .mu-sig(
        @logo-size: "48",  
        @wordmark-color: "white", 
        @print: "yes" 
    );
}

MU Signature IE8 Fallback

There is a PNG fallback mixin for browsers that do not support SVG images. When needed, use the .mu-sig-ie() mixin in your IE specific stylesheet to override the SVG image with a PNG version.

.mu-sig-ie() must be used in conjunction with the .mu-sig(). The variables below must be set again if used in .mu-sig().

@logo-size 
@logo-style 
@wordmark-color 
@print

Mizzou Logo

University of Missouri

<div class="mu-sig-ie-example">
    <p class="logo"><a href="http://missouri.edu">Mizzou Logo</a></p>
    <h1 class="wordmark"><a href="http://missouri.edu">University of Missouri</a></h1>
</div>
.mu-sig-ie-example {
    .mu-sig(
        @logo-size: "64", 
        @print: "yes" 
    );
}

// Override for IE8 only stylesheet
.mu-sig-ie-example {
    .mu-sig-ie(
        @logo-size: "64", 
        @print: "yes" 
    ); 
}
